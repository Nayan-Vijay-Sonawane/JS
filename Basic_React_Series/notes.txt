React JS: 
Its a JS library  
A single page application

- A JavaScript library for building user interfaces.
    - Build around Components
    - UI is a function of state 

    Component-Based Architecture 
        - Each piece has its own logic, is reusable, easy to maintain.
 
Project Folder Structure: 
 
    - node modules: every needed files which npm installs are in this Folder  
    - public : every static files which we needed comes under this Folder(icon)
    - src -> 
        - assets : font, image, extra css, an asset which is helpful for us in the Project
        - index.html : entry point of the application, contains root DOM
        - main.jsx : react app entry file, mounts react to the DOM(connect react to the browser)
        - app.jsx : root component (when page loads this is the first executing file) holds layout, main UI logic
    - package.json : packages we'll use there versions, dependencies are listed here.
    
JSX : 
    - Its a modern code for writing HTML and JS together
    - Browser don't understand the JSX so need to convert that into plain JS so Babel(compiler) comes in which converts that and render's also
    - JSX -> React -> HTML -> Renders on Browser  

Component : 
    - It is a reusable piece of code to define certain part of UI.
    - Functional Component is far better than class component and easy to use
    - Class Component is old and we have to use advanced JS for that(complex).

Props :
    - Data passed from parent to child component (Think props like function argument)
    - Data usually lives higher-up
    - UI pieces lives lower down
    so react needs a way to send data to down -> props   

CSS :
    - These are CSS files where class names are locally scoped to a component preventing style conflicts
    - for eg. Button.module.css
    - use class -> ClassName={styles.ClassName} 
    - Normal CSS -> Global styles(everyone can use them)
    - CSS Modules -> Private styles(only the component can use them)

Hooks: 
    - React Hooks are special functions that allow you to use state and other React features in functional components.
    - State - state is a data that the component owns and manages itself
            - Data owned by a component
            - when it changes UI updates automatically
            - state change -> re-render
    - Mount -> when a component is rendered for the first time, it is called mounting
    - Unmount -> when a component is removed from the page, it is called Unmounting

    useState Hook
    const [count, setCount] = useState(0);
        - count = Current value 
        - setCount = Function to update value
        - 0 = initial value
        (This is array destructuring)

    useEffect Hook 
    useEffect(() => {
        // Effect Logic
    }, [])
        - React calls component function
        - JSX is returned
        - DOM updates (UI appears)
        - useEffect runs 

    - Allow side effects in function component
    When runs 
        - After the first render (component mounts) 
        - After every updates (re-renders), if no dependency array is provied
        - can be controlled by passing dependency array to trigger the effect only when specific state or props change
 
    useEffect lets you run code
    - after render
    - when state change
    - when component mounts/unmounts
    Think of it as React's side-effect manager 

    useContext 
    - To consume the global state without prop-drilling. 

    useRef 
    - To hold the mutable values which do not trigger the re-render or To access the DOM
 
    useReducer 
    - For complex state management(like light version of redux)

    useMemo and useCallback
    - For optimization and to avoid unnecessary re-renders

State Lifting :
    - state lifting (lifting state up) is the process of moving state from child component to their nearest common parent so
      multiple components can share and stay synchronized with the same data through props.
    - two or more components need the same data
    - one component updates the data and another displays or uses it.

Two-way binding :
    - two-way binding means the UI updates the state and the state updates the UI (Its done using state variable and the onChange event)
    - user types -> onChange event -> setName(e.target.value) -> state updates -> input value updates

Conditional Rendering : 
    - show different UI depending on a condition 
    - automatically updates the screen when data changes

    React renders UI component based on state
    - using if/else
    - using ternery operator
    - using Logical operator (&&)
    - using early return    

Event handling :
    - Events are nothing but the actions or occurrences happen in the browser.
    - eg. click, key press, scroll, etc.
        ->  button → Event source.
            "click" → Event.
            addEventListener() → Registers the event handler.
            The function inside it executes when the button is clicked.
    - onClick, onMouse, onChange, etc.

Local Storage : 
    - local storage is a web browser feature that lets websites store small amount of data directly on a user's device
    - In the form of key-value pair
        - store a value -> localstorage.setItem = {"username", "Nayan"}
        - retrieve a value -> localstorage.getItem("username")
        - remove a value -> localstorage.removeItem("username") -> localstorage.clear()
    - The data persists even after the browser is closed, window is closed (permanent storage)

Session storage : 
    - session storage is a browser storage mechanism that stores data temporarily 
    - The data should be disappear when the tab is closed 

API calling : 
    Fetch API -> The fetch API provides an interface for fetching the resources. 

    Axios -> Axios is a popular JavaScript library used to make HTTP requests to APIs
        - sends HTTP requests and receives HTTP responses
        - axios.get() -> get request (displaying data)
        - axios.post() -> post request (sending data)
        - axios.patch() -> update parts of resource
        - axios.delete() -> delete the data
