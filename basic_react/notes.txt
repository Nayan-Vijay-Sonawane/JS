PART-2 : Virtual DOM 

    - The Problem(Old Way)
    document.queryselector("#title).innerText = "Hello";
        - manual updates
        - slow
        - bug-prone

    - React's solution
        - React creates a Virtual DOM (copy of real DOM)
        - Virtual DOM is an in-memory lightweight copy of real DOM.
        - when we update the UI, react doesn't touch the real DOM directly.
          Instead, 1. Updates the Virtual DOM first
                   2. compare it with previous Virtual DOM snapshots (Diffing)
                   3. Applies only required changes to the real DOM (reconciliation).

PART-5 : Props (Passing Data to Components)

    What are props?
    props = data passed from parent to child component. (Think props like function arguments)
    - Data usually lives higher up
    - UI pieces live lower down
    so React needs a way to send data down -> props

    Rules-
    - Props are read only(Immutable)
    - Child cannot modify props
    - Data flows one direction (top -> down)


PART-6 : State (Dynamic Data)
    
    Why State Exists?
    let count = 0;
    count = count + 1;
        - UI will not update
        - react doesn't track normal variables
        - So React introduces State
    
    What is state
        - Data owned by a component
        - When it changes -> UI updates automatically
        - State change = re-render

    useState Hook
    const [count, setCount] = useState(0);
        - count = Current value
        - setCount = Function to update value
        - 0 = initial value
        (This is array destructuring)


    Mount
    function App(){
        return<User/>
    }
    when <user/> appears on screen- It is mounted

    Unmount
    - component is removed from the DOM
    {show && <user/>}
    - If show becomes false - disappears
    That is unmounted


    useEffect Hook
    useEffect(() => {
        // Effect logic
    }, [])
        - React calls component function
        - JSX is returned
        - DOM updates (UI appears)
        - useEffect runs

        - Allow side effects in function component
        When runs
            - After the first render(component mounts)
            - After every updates(re-renders), if no dependency array is provided
            - can be controlled by passing dependency array to trigger the effect only when specific state or props change

            useEffect lets you run code
            - after render
            - when state change
            - when component mounts/unmounts
            Think of it as React's side-effect manager

PART-11 : Dependency Array 
    
    Run Once (on mount)
    useEffect(() => {
        console.log("Mounted");
    }, [])
    - API calls
    - Event listeners
    - Initial setup

    Run When State changes
    useEffect(() => {
        console.log("Count Changed")
    }, [count]);
    - Runs when count changes

    Run on multiple dependencies
    useEffect(() => {
        console.log("User or theme changed");
    }, [user, theme]);
    - Runs when one of them changes

    [] = Only once (on mount)
    [count] = when count changes
    [a, b] = when a OR b changes
    no array = on every render

PART-12 : Lifting State up

    Problem
    App
     |-CounterDisplay
     |-CounterButtons
    Two sibling components need the same data
    If both need access to count, where should state live(In the closest common parent)

    passing data down(props)
    <CounterDisplay count={count} />
    <CounterButtons count={count} setCount={setCount}/>
    - Parent sends data to children using props
    - This is called one-way data flow(top -> down)

    Parents owns the state -> children receive it via props -> children update it using the callback funcitons
    That's called Lifting State Up

PART-14 : Context API (solution to prop drilling)

    - Context API in react allows to share data accross components without manually passing props at every level
    - useful for global or app-wide state

    - Create context
    import {createContext} from 'react';
    const UserContext = createContext();

    - Provide context
    <UserContext.Provider value={user}>
        <Dashboard/>
    </UserContext.Provider>
    now, every component inside <Dashboard/> can access user

    - Consume context
    use the UserContext hook inside any child component
    import {UserContext} from 'react';
    import {UserContext} from './UserContext';
    funciton UserProfile(){
        const user = useContext(UserContext);
        return <h2>{user.name}</h2>
    }

    - useful for
        - Authentication (logged-in user)
        - Theme (dark/light mode)

PART-18 : React Router (Multi-page apps in SPA)

    - React apps are Single Page Application(SPA), they don't reload the page-we switch views

    - Wrap app in BrowserRouter
    import React from 'react';
    import ReactDOM from 'react-dom/client';
    import {BrowserRouter} from 'react-router-dom';
    import App from './app';
    ReactDOM.createRoot(document.getElementById('root')).render(
        <BrowserRouter>
        <App/>
        </BrowserRouter>
    );
    BrowserRouter enables routing for your entire application
    Without it, Route, Routes and Link won't work

    - Define Routes
    import { Routes, Route } from 'react-router-dom';
    import Home from './pages/Home';
    import Products from './pages/Product';
    function App(){
        return(
            <Routes>
                <Route path="/" element={<Home/>} />
                <Route path="/" element={<Product/>} />
            </Routes>
        )
    }
    export default App;
    URL         Component Rendered
     /             <Home/>
     /Products      <products/>
     Routes loojs at the URL and renders the matching Route
    
    - Navigation Links
    import { Link } from 'react-router-dom';
    function NavBar() {
        return(
            <nav>
                <Link to="/">Home</Link>
                <Link to="/products">Products</Link>
            </nav>
        )
    }
    export deafult NavBar;
    
    - BrowserRouter -> Enables routing
    - Routes -> Looks at current URL
    - Route -> Decides what component to render
    - Link -> Changes URL without refreshing 

    Dynamic Routes
    Route setup
    <Route path="/products/:id" element={<ProductDetails/>} /> can fetch Product using that id

    Nested Routes
    /dashboard
    /dashboard/settings
    /dashboard/Profile

    Protected Routes
    "/"
    <Route 
        path="/dashboard"
        element={
            <ProtectedRoutes>
            <Dashboard/>

        }
